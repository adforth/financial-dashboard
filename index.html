Google Sheets sync requires the hosted version
You're opening this dashboard as a local file (file://). Browser security blocks Google OAuth from working with local files — it requires HTTPS from a registered domain.

To use Google Sheets sync, open the hosted version instead:
Open adforth.github.io/financial-dashboard/
Your locally-saved data is safe — it stays in this browser's localStorage and is not affected.
Create your Google Sheet
Your financial data will be stored in a Sheet in your own Google Drive — only you can access it.
Create New Google Sheet →
After opening the Sheet, copy the Sheet ID from the URL bar — it's the long string between /d/ and /edit:

docs.google.com/spreadsheets/d/1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms/edit
The ~44-character alphanumeric string from the URL (between /d/ and /edit)
Create a Google OAuth Client ID
This lets the dashboard securely read and write your Sheet. You only need to do this once.
Open Google Cloud Console →
  1. Create a new project (or select an existing one)
  2. Click + Create Credentials → OAuth client ID
  3. Set application type to Web application
  4. Under Authorized JavaScript origins, add your dashboard URL (e.g. http://localhost:PORT or your GitHub Pages URL)
  5. Click Create, then copy the Client ID

Also enable the Google Sheets API: search for it in the API Library and click Enable.
Ends in .apps.googleusercontent.com
Authorize access
Your dashboard will request permission to read and write only your Google Sheets. No other Google data is accessed. You can revoke access at any time from your Google Account settings.
Click Connect with Google below. A Google sign-in window will open — sign in and approve access to Google Sheets.
